# Benchmark 设计需求

## 背景

结合 PaperBench 和 PaperWritingBench，设计适合 MinionsOS2 的科学发现 benchmark。

## 核心需求

### 1. 任务内容
- 基于 PaperWritingBench 的文档材料
- 从现有材料出发，提出更好的方法
- 开放式任务设计

### 2. 场景选择（三个代表性场景）
- **理论研究**: 偏数学证明、算法分析
- **简单 ML**: 如分类、回归任务
- **深度学习**: CNN、Transformer 等

### 3. 设计原则
- **前沿性**: 选择主流、前沿主题
- **特色**: 体现 MinionsOS2 的 Ethics 审核特点
- **可执行**: 考虑计算资源限制

### 4. 评价维度
- 性能对比（横向比较）
- 多元评估（创新性、可行性、伦理性等）

## 输出要求

团队讨论后输出一份完整的 **Proposal**，包含：
- Benchmark 整体设计
- 三个场景的具体任务定义
- 评价标准和实施方案
